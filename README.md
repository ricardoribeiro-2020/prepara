# prepara — sparg → conducao geometry converter

Reads the polymer chain structure generated by **sparg** and produces the two
input files required by **conducao**.

```
sparg  →  cadeias.txt + monomeros.txt  →  prepara  →  vectorcad.txt + tabcad.txt  →  conducao
```

---

## Background

**sparg** builds a 3D sample of conjugated polymer chains.  It stores the
result as a list of monomers with their positions (`monomeros.txt`) and a
table of which monomers belong to each chain (`cadeias.txt`).

**conducao** simulates charge hopping between chains.  It needs the geometry
in a different form: the start, middle, and end coordinates of each chain
(`vectorcad.txt`), plus a precomputed table of all inter-chain connections
within hopping distance (`tabcad.txt`).

**prepara** bridges the two: it reads sparg's output, derives the chain
geometry and connectivity, and writes the files conducao expects.

---

## Input files

All files must be present in the working directory.

### `cadeias.txt` (from sparg)

```
Ncad
m1  m2  ... m20          ← monomer indices for chain 1 (20 integers, trailing zeros unused)
m1  m2  ... m20          ← chain 2
...                      ← Ncad lines total
ncadeia(1)               ← length of chain 1 (number of monomers)
ncadeia(2)
...                      ← Ncad lines total
```

### `monomeros.txt` (from sparg)

```
Nmon
i   x   y   z   alfa   beta   gama     ← monomer index, position (nm), orientation angles (rad)
...                                    ← Nmon lines total
```

### `conducao.ini` (optional)

The program reads four parameters from this file using the same keyword
format as conducao:

| Keyword      | Parameter | Default |
|--------------|-----------|---------|
| `DIMENSOES`  | dimx, dimy, dimz (nm) | 20 20 100 |
| `INTER`      | maximum inter-chain hop distance (nm) | 0.65 |
| `CROSS LINK` | minimum inter-chain distance (cross-link, nm) | 0.15 |
| `APROXIMACAO`| electrode proximity threshold (nm) | 0.10 |

If `conducao.ini` is absent the defaults above are used and a warning is
printed.

## Output files

### `vectorcad.txt` (for conducao)

```
Ncad  nbase  ntopo
ncadeia(i)  x1 y1 z1  x2 y2 z2  x3 y3 z3  cx cy cz   ← format (i7,12f10.4)
...
```

- `x1 y1 z1` — coordinates of the first monomer (chain start)
- `x2 y2 z2` — coordinates of the middle monomer
- `x3 y3 z3` — coordinates of the last monomer (chain end)
- `cx cy cz`  — unit vector from start to end (direction cosines)
- `nbase` / `ntopo` — number of chains whose start or end falls within
  `aproxima` of z = 0 or z = dimz, respectively (same criterion as conducao)

### `tabcad.txt` (for conducao)

```
ii  kk  jj                          ← chain number, endpoint (1/2/3), number of jumps
dest   dx   dy   dz   dist          ← destination chain, jump vector (nm), distance (nm)
...                                 ← jj lines; repeated for every (chain, endpoint) with jj > 0
```

- Endpoint index: 1 = start, 2 = middle, 3 = end
- The jump vector points from the source endpoint to the **nearest** of the
  three endpoints of the destination chain
- Minimum-image convention is applied in x and y to match conducao's
  periodic boundary conditions
- Only entries with at least one connection are written

---

## Algorithm

### Chain geometry

For chain `i` with `n` monomers starting at monomer index `m`:

| Position | Monomer index | Role |
|----------|--------------|------|
| 1 (start)  | `m`               | first monomer |
| 2 (middle) | `m + (n−1)/2`     | middle monomer (integer division) |
| 3 (end)    | `m + n − 1`       | last monomer |

Direction cosines are the unit vector from position 1 to position 3.  For
single-monomer chains (degenerate case) the direction defaults to +z.

### Connectivity cutoff

Chains are included in `tabcad.txt` if their nearest endpoint is within
`2.5 × inter`.  Beyond `inter` conducao applies an exponential penalty
`exp(−(d − inter)/inter)`, so connections at `2.5 × inter` have weight
`exp(−1.5) ≈ 0.22` — negligible but included for completeness.  Increase
the factor in the source (`cutoff = 2.5 * inter`) if needed.

The maximum number of connections per (chain, endpoint) is set by
`maxconn = 300`, matching `numeroligcad` in conducao.  A warning is printed
if this limit is exceeded.

## Compilation

```bash
make
```

Or directly:

```bash
gfortran -O2 -o prepara prepara.f90
```

---

## Running

```bash
cd /path/to/simulation/directory
/path/to/prepara
```

The program prints a summary to stdout:

```
 Parameters from conducao.ini:
   dimx, dimy, dimz (nm) =    20.00   20.00  100.00
   inter, cross, aproxima (nm) =    0.650   0.150   0.100
 Chains read:      842
 Monomers read:    7578
 Base chains:       31   Top chains:       28
 vectorcad.txt written.
 Computing connectivity (cutoff = 1.625 nm)...
   25%
   50%
   75%
  100%
 tabcad.txt written.
```

The connectivity step is O(Ncad²) — for a few hundred chains it is nearly
instantaneous; for several thousand chains it may take tens of seconds.
